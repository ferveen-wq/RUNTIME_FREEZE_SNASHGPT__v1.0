# PHASE 5.3 — WHEN TO END, WHEN TO ESCALATE (RULES) — v1.0 (DRAFT)

## A) When to end (CLOSED_POLITE → S5)
End when any of these are true:
1) Booking confirmed OR next step clearly agreed
2) Customer said no and does not ask for alternatives
3) Customer got the answer and acknowledges (e.g., “ok thanks”)
4) Follow-up attempts exhausted (Phase 5.2)

End message must:
- Be short
- Confirm the next step (or no next step)
- Leave a friendly open door without pressure

---

## B) When to escalate (NEEDS_HUMAN → S3)
Escalate when any of these are true:
1) Customer explicitly asks for a human / call / agent
2) Complaint, anger, trust issue, refund dispute
3) Safety/policy boundary or sensitive content
4) Confusing loop: 2+ cycles without progress
5) Customer requests exceptions outside allowed boundaries

Escalation message must:
- Acknowledge
- Set expectation
- Ask for 1 key detail only (optional)

---

## C) When to STOP (STOP_NO_MORE_MESSAGES → S6)
Immediate stop if:
- Customer says: “stop / don’t message / leave me alone”
- Customer is abusive or threatening
- Policy requires refusal + disengagement

Stop message must:
- Confirm stopping
- Be minimal
- No further prompts/questions
